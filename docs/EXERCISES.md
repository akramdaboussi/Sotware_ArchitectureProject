# 🎯 Hands-On Exercises

> **Goal**: Master the authentication system with H2/JPA in 15-20 minutes. 8 progressive exercises.

---

## 🏃 Express Track (18 minutes)

### Exercise 1: **Explore the API** ⏱️ 2 min
**Discover all available endpoints**

Open `AuthController.java` and `AdminController.java`, list all endpoints:

**Auth** (`/api/auth/`):
- ☐ `POST /register` → Registration
- ☐ `POST /login` → Login
- ☐ `POST /logout` → Logout
- ☐ `GET /me` → Current user info

**Admin** (`/api/admin/`):
- ☐ `GET /users` → List all users
- ☐ `POST /add-role` → Add role to user
- ☐ `POST /remove-role` → Remove role from user

**Quick Test**: Go to http://localhost:8080 and register a new user!

---

### Exercise 2: **Explore the H2 Database** ⏱️ 3 min
**See persisted data**

1. Go to http://localhost:8080/h2-console
2. Connection settings:
   - JDBC URL: `jdbc:h2:file:./data/authdb`
   - Username: `sa`
   - Password: *(empty)*
3. Click "Connect"

**Queries to try**:
```sql
-- See all users
SELECT * FROM USERS;

-- See all roles
SELECT * FROM ROLES;

-- See roles by user
SELECT u.EMAIL, r.NAME FROM USERS u 
JOIN USER_ROLES ur ON u.ID = ur.USER_ID
JOIN ROLES r ON ur.ROLE_ID = r.ID;

-- See active tokens
SELECT * FROM TOKENS WHERE REVOKED = FALSE;
```

**Question**: Which roles were automatically created at startup?
**Answer**: ROLE_USER, ROLE_ADMIN, ROLE_MODERATOR (see `DataInitializer.java`)

---

### Exercise 3: **Trace the Registration Flow** ⏱️ 2 min
**Understand the password path**

1. `AuthController.register()` → receives plain password
2. `AuthService.register()` → calls `userService.hashPassword()`
3. `UserService.hashPassword()` → SHA-256 + Base64
4. `userRepository.save(user)` → INSERT into H2

**Check in H2**:
```sql
SELECT EMAIL, PASSWORD FROM USERS;
```
→ Is the password readable? **No!** It's hashed.

---

### Exercise 4: **Understand JPA Repositories** ⏱️ 2 min
**See how JPA simplifies data access**

Open `UserRepository.java`:
```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);
    boolean existsByEmail(String email);
}
```

**Questions**:
1. Where is the SQL code for `findByEmail()`?
   → **Answer**: Auto-generated by JPA!
2. What does `JpaRepository<User, Long>` provide?
   → **Answer**: Free save(), findById(), findAll(), delete() methods

---

### Exercise 5: **Test Admin Roles** ⏱️ 3 min
**Experiment with access control**

**Step 1**: Create an admin manually (H2 Console)
```sql
-- Find the ADMIN role ID
SELECT * FROM ROLES WHERE NAME = 'ROLE_ADMIN';

-- Add the role to your user (replace IDs)
INSERT INTO USER_ROLES (USER_ID, ROLE_ID) VALUES (1, 2);
```

**Step 2**: Test the admin endpoint
```bash
# With your token (retrieved after login)
curl -X GET http://localhost:8080/api/admin/users \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

**Step 3**: Test WITHOUT being admin
- Create a new user → try to access `/api/admin/users`
- **Expected result**: `403 Forbidden - Access denied: Admin role required`

---

### Exercise 6: **Watch Logs in Real-Time** ⏱️ 2 min
**Understand what happens with each request**

1. Look at the terminal running `mvn spring-boot:run`
2. Register a new user from the UI
3. Watch the log tags:
   - `[INIT]` → Role initialization at startup
   - `[CONTROLLER]` → HTTP request received
   - `[SERVICE-AUTH]` → Authentication logic
   - `[SERVICE-USER]` → Password hashing
   - `[SERVICE-TOKEN]` → Token generation

**Bonus**: Look for `Hibernate:` in logs → these are generated SQL queries!

---

### Exercise 7: **Debug with a Breakpoint** ⏱️ 3 min
**Inspect variables in real-time**

1. In VS Code, open `AuthService.login()`
2. Click to the left of line `User user = userService.findByEmail(email);`
3. Start debugging (F5) or attach to server
4. Login from the UI
5. When breakpoint activates, inspect:
   - `email` → The entered email
   - `user` → User object with its roles
   - `user.getRoles()` → Set of associated roles

**Discovery**: Look at `user.getPassword()` → it's the hash, not the password!

---

### Exercise 8: **Add a New Role** ⏱️ 3 min
**Modify the DataInitializer**

1. Open `config/DataInitializer.java`
2. Add a new role in the `run()` method:
```java
createRoleIfNotExists("ROLE_VIP", "VIP user with special access");
```
3. Save → Server restarts automatically
4. Check in H2:
```sql
SELECT * FROM ROLES;
```
→ You should see `ROLE_VIP`!

## 🎓 What You Learned

✅ **Architecture**: Controllers → Services → Repositories → Database (H2)  
✅ **JPA**: Auto-generated repositories, no manual SQL  
✅ **Security**: Hashed passwords (SHA-256), JWT tokens (stateless, signed), RBAC  
✅ **Roles**: ROLE_USER, ROLE_ADMIN, ROLE_MODERATOR + custom  
✅ **H2 Console**: Explore data in real-time  
✅ **Debugging**: Layered logs + VS Code breakpoints
